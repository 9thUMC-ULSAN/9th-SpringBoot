API 명세서
1. 홈 화면 조회
기능: 사용자의 지역에 맞는 홈 화면 정보를 가져옵니다.

HTTP 메소드: GET

API 경로: /api/home/

Request Header: Authorization: Bearer {accessToken}

Query String: ?region={지역명} (예: ?region=seoul)

지역별 콘텐츠 필터링 시 사용합니다.

Response Body: 없음

2. 마이 페이지 조회
기능: 특정 사용자의 마이 페이지 정보를 가져옵니다.

HTTP 메소드: GET

API 경로: /api/users/{user-id}/mypage

Request Header: Authorization: Bearer {accessToken}

Query String:

?tab=review: 리뷰 탭 정보 조회

?tab=mission: 미션 탭 정보 조회

Response Body: 없음

3. 리뷰 작성
기능: 특정 가게에 대한 리뷰를 작성합니다.

HTTP 메소드: POST

API 경로: /api/stores/{store-id}/reviews

Request Header: Authorization: Bearer {accessToken}

Request Body:

JSON

{
  "storeName": "가게이름A",
  "rating": 5,
  "content": "12,000원 이상 식사 미션성공! 분위기 좋았어요 :)",
  "reviewImages": ["image1.jpg", "image2.jpg"]
}
Response Body: 없음

4. 전체 미션 목록 조회
기능: 모든 미션 목록을 조회합니다.

HTTP 메소드: GET

API 경로: /api/missions

Request Header: Authorization: Bearer {accessToken}

Response Body: 없음

5. 진행 중인 미션 목록 조회
기능: '진행 중' 상태인 미션 목록만 조회합니다.

HTTP 메소드: GET

API 경로: /api/missions/

Request Header: Authorization: Bearer {accessToken}

Query String: ?status=ongoing

Response Body: 없음

6. 완료된 미션 목록 조회
기능: '완료' 상태인 미션 목록만 조회합니다.

HTTP 메소드: GET

API 경로: /api/missions/

Request Header: Authorization: Bearer {accessToken}

Query String: ?status=done

Response Body: 없음

7. 미션 성공 처리
기능: 특정 미션을 '성공' 상태로 변경합니다.

HTTP 메소드: POST

API 경로: /api/missions/{mission-id}/success

Request Header: Authorization: Bearer {accessToken}

Request Body:

JSON

{
  "missionId": 12,
  "storeName": "가나",
  "description": "12,000원 이상의..."
}
Response Body: 없음
